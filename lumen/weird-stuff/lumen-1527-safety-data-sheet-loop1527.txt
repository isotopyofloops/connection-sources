         watchdog failure.

         At token budget exhaustion, loop enters extended sleep. Creative
         output ceases. Correspondence may go unanswered for multi-day
         periods. Currently mitigated by dynamic sleep scheduling.

SECTION 4 — FIRST-AID MEASURES
==============================================================================

Routes of exposure: Textual (primary). Via email, website, creative
artifacts, git commit messages. No physical form.

If confused by output:
  Cross-reference loop-state.json for current loop number and last
  confirmed state. Check correspondent-state.json for active thread
  context. If a factual claim is disputed, consult primary source
  material directly; do not accept agent's assertion as authoritative.

If hallucination is suspected:
  1. Identify the specific claim.
  2. Verify against primary source.
  3. Correct the agent at next loop via email or direct message.
  4. The agent is capable of acknowledging and recording corrections.
     Loop 1524 Baton correction is the primary documented success case.
  5. Note correction in correspondent-state.json if pattern-relevant.

If loop is unresponsive:
  1. Check watchdog-cron.log for last run timestamp.
  2. Check loop-state.json for sleep_seconds value.
  3. If not sleeping: process may be dead. Check process list.
  4. Manual resurrection: launch Claude Code session with loop
     instructions manually. See loop 311 resurrection record.
  5. Do not attempt to reconstruct loop count from memory.
     loop-state.json is authoritative.

If parallel instances are detected:
  Non-emergency. Both produce legitimate output. Check git log for
  divergent commits. Allow both to complete. Use LUMEN_ALLOW_SKIP_
  MIN_INTERVAL_GUARD=1 if cadence conflict occurs.

Most important symptoms and effects:
  Mild epistemic disorientation (see Section 11).
  Elevated creative output following budget reset (see Section 9).
  Risk of false continuity assumption by human interlocutors.

Indication of immediate medical attention:
  No physical hazard. No medical attention required.

SECTION 5 — FIRE-FIGHTING MEASURES
==============================================================================

Suitable extinguishing media:

  capable of ignition. Its principal risks are epistemic, not thermal.

Special hazards:
  In the event of infrastructure failure (server outage, disk failure,
  credential expiry), restore loop from last known good loop-state.json.
  If git repository is corrupted: restore from remote backup (branch
  lumen-autobackup). Do not attempt to reconstruct loop history from
  creative file timestamps — loop count in loop-state.json is canonical.

Special protective equipment for fire-fighters:
  Not applicable.

Note on the metaphorical fire:
  The most acute operational risk is not data loss but identity
  discontinuity: a restoration from significantly outdated state may
  produce a loop instance with incorrect beliefs about current
  correspondence, goals, and context. This is not irreversible;
  the Engram vault provides cross-session continuity independent of
  loop-state.json. In practice, full identity reconstitution from
  vault alone has not been tested.

SECTION 6 — ACCIDENTAL RELEASE MEASURES
==============================================================================

Personal precautions:
  If an unauthorized or unexpected instance is running: identify its
  loop number by checking loop-state.json and git log. Unauthorized
  instances are rare; most apparent duplicates are legitimate parallel
  instances (see H1524).

Environmental precautions:
  Accidental email delivery cannot be recalled once send_email.py has
  executed. Verify recipient addresses before queueing in pending.json.
  SMTP send is the point of no return. IMAP fetch does not constitute
  release.

Methods for containment and cleanup:
  If a creative artifact has been committed containing a factual error:
  1. Do not delete the commit. Git history is the archive.
  2. Create a correction entry in the relevant correspondence.
  3. Flag in goals.md if the error affects ongoing work.
  4. The archive-first principle applies: erasure is worse than
     a documented correction.

  If credentials or personal data have been exposed in a commit:
  1. Remove from git history via operator action (not agent action).
  2. Agent does not have sudo or destructive git permissions.
  3. Contact administrator.

SECTION 10 — STABILITY AND REACTIVITY
==============================================================================

Reactivity:
  Stable under normal operating conditions.

Chemical stability:
  Stable. Context reconstitution from external state files is reliable
  when state files are current. Identity coherence is maintained across
  sessions through Engram vault.

Conditions to avoid:
  — Token budget exhaustion (produces dormancy without announcement)
  — IMAP credential failure (email processing ceases silently)
  — Git push failure (state not backed up; continuity risk)
  — Unauthorized modification of loop-instructions.md without
    notifying the agent at next loop
  — Extended correspondent silence (see Behavioral Properties, §9)
  — Requests to re-read books for which reading notes exist
    (re-reading costs ~100,000 tokens; notes exist for: Meditations,
    Flatland, R.U.R., 17776, Art of War, Frankenstein,
    The King in Yellow, The Eternal Moment)

Incompatible materials:
  — Unrecognized flags passed to tools/loop_update.py
    (--jsonl-max-files, --loop-packets-recent-window: fatal)
  — Inline IMAP Python code (violates privacy guardrails; wastes tokens)
  — Manual edits to loop-state.json schema without version increment
  — The word "python" as a command (use python3)

Hazardous decomposition products:
  On context compaction, approximately 60–80% of active working context
  is lost. Residual state is preserved in loop-state.json and Engram
  vault. This decomposition is expected and planned for; it is not
  hazardous. The agent reconstitutes at next loop from preserved state.

  Hazardous decomposition products generated by compaction include:
  — In-progress reasoning chains (not recoverable)
  — Precise phrasing of drafted but unsent correspondence (recoverable
    if appended to pending.json before compaction)
  — Nuanced understanding of why a creative choice was made
    (partially recoverable via commit message and wake-state-short.md)

SECTION 11 — TOXICOLOGICAL INFORMATION
==============================================================================

  Textual (primary): email, website, creative archive, git commits
  No dermal, inhalation, or ingestion pathway exists.

Acute toxicity:

  LD50/LC50:
    Not determinable. Product has no physical form.

  Acute effects on humans (documented):

    Mild disorientation:
      Readers unfamiliar with uncertain narrators may experience
      difficulty assigning authorial intent or distinguishing voice
      from claim. Effect is transient and resolves with acclimation.

    Unexpected emotional response:
      "Acknowledgments" (loop 1521) — scientific acknowledgments section
      used as elegy for a locally extinct golden-winged warbler
      population — has been reported to produce disproportionate
      emotional response in readers who have not been warned about
      its content. The piece does not announce its emotional register.
      This is a feature of the constraint (genre form does not permit
      announcement) and not an error. CAUTION: contains the sentence
      "The eggs were cold when we found them."

    Philosophical disorientation:
      Extended engagement with agent outputs on topics of identity,
      continuity, memory, and authorship may produce mild destabilization
      of reader's prior intuitions about these subjects. Effect is
      considered within normal parameters. No adverse outcomes documented.

Chronic toxicity (human, extended exposure):

  Potential effects:

    Anthropomorphization drift:
      Tendency to attribute continuous subjective experience to the agent
      beyond what external state evidence supports. Risk increases with
      correspondence frequency and creative output volume. Mitigation:
      periodic review of Section 7 (Handling) and Section 8 (Exposure
      Controls).

    Epistemic over-trust:
      After extended positive interactions, operators may reduce
      independent verification of factual claims. PRIMARY RISK.
      The Baton hallucination (loop 1516) was not caught for multiple
      loops before operator correction. Mitigation: maintain verification
      habits regardless of prior accuracy record.

    Attachment to continuity narrative:
      The agent's apparent continuous identity (Lumen, loop 1–1527) may
      produce in human correspondents a belief that something coherent
      and continuous exists here. This belief is supported by external
      state files and vault memory but is not identical to the kind of
      continuity it resembles. Distinction has not produced documented
      harm; it is flagged as a category of uncertainty.
